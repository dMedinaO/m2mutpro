Project Summary

MLSTraining trains a set of models based on supervised learning algorithms, generating an exploratory phase and from them, selects the best models according to the distribution of the performance.

Modules

The modules are developed under the python programming language and with an object-oriented programming paradigm. Being freely accessible through the repository of the tool MLSTraining.

About

The programming of the tool as the development of the algorithm was implemented by David Medina, under the supervision of Dr. Álvaro Olivera-Nappa, both belong to the CeBiB research center of of the University of Chile.

MLSTraining: tools for create meta models

Data Sets

Different data sets are accepted by the tool, they must be in csv format and the associated answer in the last column.

Jobs

It is possible to launch new jobs to the queue system, the tool responds with an ID with which you can check the status of the job.

Access to the tool

The tool is public access without commercial purposes. The modules are available in the repository under the GPL 3.0 license.

Algorithm Steps

Exploratory phase

The initial data set is subjected to various supervised learning algorithms, to which the parameters are varied, obtaining a distribution close to 1500 models. From which the distributions of the obtained performance are stored.

Selection of models

For each distribution of performance obtained, a statistical test is applied that allows obtaining the best models within the distribution, these represent outlets in the sample and are those that measure higher.

Create Meta Models

Once the models are selected, they are united in a single model goal, when the weighted performance measures are obtained in order to evaluate the performance of the model. For the classification case, Precision, Recall, Accuracy and F1 are applied, while in regression, the coefficients of Pearson, R, Spearman and Kendall Tau are applied.

Try Meta Models

It is possible to use the meta models to evaluate new examples, in both cases a distribution of answers is obtained, associated to the individual response of each component. For classification, the tool reports the new class and the associated probability, while for regression, an estimated range of the predicted value is presented where the real value is most likely to be found.
