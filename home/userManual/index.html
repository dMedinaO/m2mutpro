To create a new job, you must upload a file in csv format or derivatives and complete the form associated with the Job. In the case of analyzing responses with continuous variables, you must select regression models, in case the answer is categorical, select the classification option. For both cases, the system recognizes as the last column the one with the response for model training. If the system correctly validates the form, a new Job is created and launched in the queue system, an ID is given to recognize it and it is notified by mail when the job changes state.


To consult for a Job, enter the ID in the search engine, the system will deliver the state in which the Job is located. These states can be: START, PROCESSING, OBSOLETE, ERROR or FINISHED. In the case that it is finalized, the system redirects to the results of the Job. It is mentioned that the OBSOLETE state alludes to the fact that the Job exists but is not available given that the time limit was reached to visualize the results (15 days). If you wish to recover a Job, please contact david.medina@cebib.cl indicating the reason.


The visualization of the meta models depends on the type of trained response (classification or regression). In both cases, a diagram is shown with the algorithms selected for each performance measure. Following the diagram, a panel with the weighted measures is generated, which depends on the type of response. For the classification cases, Accuracy, Accuracy, Recall and F1 are used, otherwise, Pearson, R score, Kendall Tau and Spearman. Finally, for regression models, error graphs and scatter plot of the prediction v / s are appreciated, while for classification models a matrix of confusion associated with the process is exposed.


In addition to the visualization of the meta models, the performance measures of the selected models are exposed, continuing with a statistical summary of the process and histograms for each performance measure. Finally, there is a section with a summary of the process, in which the number of iterations developed, execution time and a brief description of performance measures and how to interpret them are exposed.


The tool has two options for the use of meta models trained for the classification or prediction of new examples. The first option is through the download of the meta models and have the python tool available in the project repository, execute the corresponding script with the necessary parameters, it is a way to run stand alone models and test them locally. In the case of not having knowledge in python programming or Linux terminal management, there is the web execution option, for it, at the moment of selecting it, it will be redirected to a form where it is necessary to upload the file to do the processing, the tool use the meta models stored for the job ID and deliver the results with respect to the new examples. This option may take several minutes.


The results of classification or prediction of new examples depend on the type of task to be performed. However, in both results it is a summary table where each row corresponds to an example. In the case of regression models, statistics of the values obtained are available and the value of the prediction is associated to a range where the real value is more likely to be found. In the case of classification models, the class or category to which the new example corresponds and associated with it the probabilities of the distribution of the classes generated by the classifiers for each example.
